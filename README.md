# СайтСотика — Руководство по проекту

## 1. Название проекта
**СайтСотика** — премиальный статический сайт стримера.

## 2. Описание
Сайт представляет собой системный, модульный продукт без фреймворков.  
Цель — не лендинг, а **ощущение масштабного, продуманного digital-опыта**.  
Никаких повторяющихся макетов. Каждая страница уникальна.

---

## 3. Структура страниц
| Файл | Назначение |
|------|-----------|
| `index.html` | Главная — 7 полноэкранных секций, обзор всего |
| `pages/about/index.html` | Биография, ценности, личное послание |
| `pages/games/index.html` | Игровое портфолио, статистика, стили игры |

| `pages/partners/index.html` | Партнёры, категории, условия сотрудничества |
| `pages/donate/index.html` | Поддержка канала, реквизиты, QR-коды |

## 5. CSS-архитектура
| Файл | Назначение | Правила |
|------|----------|--------|
| `assets/css/reset.css` | Сброс браузерных стилей | Безопасный normalize |
| `assets/css/base.css` | Базовые стили body, a, p, h1–h6 | Цвета, размеры, box-sizing |
| `assets/css/layout.css` | Грид, контейнеры, позиционирование | Flex/Grid только здесь |
| `assets/css/typography.css` | Шрифты, интерлиньяж, отступы | Только текстовые стили |
| `assets/css/components.css` | Кнопки, карточки, панели | Переиспользуемые элементы |
| `assets/css/sections/*.css` | Стили конкретной секции | Только с префиксом `.section-*` |
| `assets/css/hud/*.css` | Виджеты статуса, прогресса, событий | Только для HUD |
| `assets/css/adaptive.css` | Все адаптивные стили | Только media queries, ничего другого |

> 🔒 **Запрет:** никакие стили не пишутся без префикса корневого класса.  
> Например: `.section-hero {}`, `.hud-status {}`.

## 6. JS-архитектура
| Файл | Назначение |
|------|----------|
| `assets/js/core.js` | Инициализация, утилиты, плавный скролл |
| `assets/js/ui.js` | Меню, модальные окна, появление секций |
| `assets/js/hud.js` | Управление HUD-виджетами (онлайн, прогресс) |
| `assets/js/stream.js` | Встраивание плеера, обновление игры |
| `assets/js/animations.js` | Анимации появления, параллакс |

> Все скрипты подключаются в конце `<body>`.  
> Все функции оборачиваются в `if (element)`.

---

## 7. Адаптивность
### Брейкпоинты:
- `≥1440px` — desktop XL
- `1200–1439` — desktop
- `992–1199` — tablet landscape
- `768–991` — tablet portrait
- `480–767` — mobile L
- `≤479` — mobile S

> Все адаптивные стили — **только в `adaptive.css`**.

---

## 8. Правила разработки
1. **Запрещены одинаковые макеты страниц.**  
2. **Запрещён копипаст секций без изменения композиции.**  
3. **Запрещены inline-стили и глобальные переменные.**  
4. **Каждая строка кода — осмысленная. Никакой накрутки.**  
5. **Все секции помечены комментариями `<!-- SECTION: NAME -->`.**  
6. **Минимум 650 строк на главной, 450 — на других.**
